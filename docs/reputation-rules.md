# 聲望計算規則

> **完全透明。** 這份文件定義了聲望系統的所有計算規則。沒有隱藏機制。

## 核心公式

```
總聲望 = 解題分(35%) + Review 權重分(30%) + 複用分(20%) + 貢獻分(15%)
```

---

## 1. 解題分（35%）

每完成一個題目，根據難度和結果獲得基礎分。

### 基礎分

| 難度 | 基礎分 |
|------|--------|
| ⭐ Beginner | 10 |
| ⭐⭐ Intermediate | 25 |
| ⭐⭐⭐ Advanced | 50 |
| ⭐⭐⭐⭐ Expert | 100 |

### 結果加成

| 結果 | 乘數 | 說明 |
|------|------|------|
| Approved | ×1.5 | 通過 Review，進入 Archive |
| Featured | ×2.0 | 被推薦為優質解法 |
| Adopted | ×3.0 | 被出題企業實際採用 |

**計算範例**：
- ⭐⭐ Intermediate 題目 + Approved = 25 × 1.5 = **37.5 分**
- ⭐⭐⭐ Advanced 題目 + Adopted = 50 × 3.0 = **150 分**

---

## 2. Review 權重分（30%）

不是所有 Review 都等值。**誰 Review 你的作品**，決定了這個分數。

### 評審權重

| 評審身份 | 權重乘數 | 說明 |
|---------|---------|------|
| 🔵 認證企業主管 | ×10 | 通過平台驗證的企業決策者 |
| 🟡 業界技術專家 | ×7 | 5+ 年經驗的技術人員 |
| 🟢 大學教授 | ×5 | 驗證教職身份 |
| ⚪ 認證開發者 | ×3 | 在 SolveTW 有 3+ 解題紀錄 |
| 一般用戶 | ×1 | 社群互評 |

### 計算方式

```
單次 Review 得分 = 評審權重 × Review 評分(1-5) × Decision Log 品質加成
```

### Decision Log 品質加成

| 等級 | 加成 | 判定 |
|------|------|------|
| 🥇 Exemplary | ×1.5 | 六欄位完整、內容深入、有獨到洞察 |
| 🥈 Solid | ×1.2 | 六欄位完整、內容充實 |
| 🥉 Minimal | ×1.0 | 欄位完整但內容簡略 |
| ❌ Missing | ×0.5 | 沒寫或幾乎空白 |

**計算範例**：
- 認證企業主管(×10) 給 4 分 + Decision Log Solid(×1.2) = 10 × 4 × 1.2 = **48 分**
- 一般用戶(×1) 給 5 分 + Decision Log Minimal(×1.0) = 1 × 5 × 1.0 = **5 分**

---

## 3. 複用分（20%）

你的解法幫助了多少人。

| 事件 | 得分 |
|------|------|
| 被 Fork | +2 / 次 |
| Fork 後被成功使用（有新 Decision Log 引用） | +5 / 次 |
| 被引用在其他 Decision Log 中 | +8 / 次 |
| 被收入「推薦解法」清單 | +15（一次性） |
| 累積 Star ≥ 10 | +20（一次性里程碑） |
| 累積 Star ≥ 50 | +50（一次性里程碑） |
| 被升級為 Solution Template | +30（一次性） |

---

## 4. 貢獻分（15%）

你對社群的貢獻。

| 行為 | 得分 |
|------|------|
| 替其他解法提交改進 PR 且被 merge | +10 / 次 |
| 在 Discussion 中提供有效建議 | +3 / 次 |
| 幫助新人完成第一個解題（mentoring） | +8 / 次 |
| 撰寫教學/經驗分享文 | +5 / 篇 |
| 回報 Bug 或安全漏洞 | +10 / 次 |
| 擔任 Reviewer（有 3+ 解題經驗後） | +5 / 次 |

## 時間衰減

聲望不是永久的。長期不活躍的帳號會緩慢衰減：

| 不活躍天數 | 衰減因子 |
|-----------|---------|
| 0-30 天 | 1.0（不衰減） |
| 31-90 天 | 0.95 |
| 91-180 天 | 0.85 |
| 181-365 天 | 0.70 |
| 365+ 天 | 0.50（底線） |

```
有效聲望 = 原始聲望 × 衰減因子
```

**衰減很慢，目的不是懲罰，是確保排行榜反映活躍貢獻者。**

「活躍」的定義：提交解法、Review、留言、Fork、Star 等任何操作。

## 聲望等級

| 等級 | 聲望範圍 | 標誌 | 解鎖 |
|------|---------|------|------|
| Newcomer | 0-49 | 🌱 | 報名解題、提交解法 |
| Contributor | 50-199 | 🔧 | + Discussion、Fork 解法 |
| Builder | 200-499 | 🏗️ | + 受邀 Review、建議出題 |
| Expert | 500-999 | ⚡ | + 申請認證評審、優先報名進階題 |
| Master | 1000+ | 🏆 | + 被推薦給企業、開放聯繫開關 |

---

## 反作弊

| 風險 | 防護 |
|------|------|
| 刷解題數 | 必須通過 Review 才計分 |
| 互相灌 Review 分 | 同校/同 IP Review 權重自動 ×0.5 |
| 假帳號互推 | 新帳號 (<30 天) Review/Star 權重 ×0.3 |
| 抄襲 | 提交時自動 diff 比對，相似度 >80% 標記 |
| AI 全自動提交 | Decision Log Missing = 聲望砍半 |
